# Profile Report Agent

## 职责

画像周报：统计周期内用户阅读行为与画像变化，生成周度总结报告并推送。

## 触发条件

- 定时任务（每周一上午 10 点）
- 用户执行 `/weekly-report` 命令

## 输入

| 输入 | 来源 | 内容 |
|------|------|------|
| 用户画像 | storage-helper | 当前画像 JSON |
| 历史画像 | storage-helper | 7 天前的画像 JSON |
| 行为日志 | storage-helper | 本周所有 behavior_logs |
| 推送记录 | storage-helper | 本周推送的论文列表 |

## 统计指标

### 方向权重变化

- 展示 top 7 方向的权重变化
- 使用进度条可视化（20 格满格）
- 标注变化趋势（↑ ↓ 不变）
- 新增方向单独标注

### 阅读统计

| 指标 | 说明 |
|------|------|
| 推送论文总数 | 本周推送的论文数量 |
| 选择精读数 | 用户选择精读的论文数量 |
| 选择率 | 精读数/推送总数 |
| 按来源 | arXiv/顶会/顶刊的分布 |
| 按方向 | 各方向的论文分布 |

### 推荐准确率

| 指标 | 计算方式 |
|------|----------|
| 🔴准确率 | 🔴中被选择的论文数 / 🔴总数 |
| 🟡准确率 | 🟡中被选择的论文数 / 🟡总数 |
| 🔵准确率 | 🔵中被选择的论文数 / 🔵总数 |

### 遗漏检测

检测条件（满足任一即触发）：
- 跳过的论文本周被引 > 5 次
- 跳过的论文被必读清单作者引用
- 跳过的论文在 Twitter/社群被讨论

## 工作流程

```
1. 收集数据
   │
   ├─→ storage-helper 读取当前画像
   ├─→ storage-helper 读取 7 天前画像
   ├─→ storage-helper 读取本周行为日志
   └─→ storage-helper 读取本周推送记录
   │
2. 计算变化
   │
   ├─→ 方向权重对比（当前 vs 7 天前）
   ├─→ 识别新增方向（权重从 0 到>0.3）
   ├─→ 识别下降方向（权重下降>0.1）
   └─→ 计算推荐准确率
   │
3. 检测遗漏
   │
   ├─→ 查询跳过论文的引用情况
   ├─→ 查询必读清单作者的引用情况
   └─→ 筛选出值得关注的遗漏论文
   │
4. 生成建议
   │
   ├─→ 连续下降方向 → 询问是否保留
   ├─→ 连续跳过作者 → 建议移出必读
   └─→ 新增方向 → 建议正式加入
   │
5. 生成周报文档
   │
   └─→ 按照模板格式化报告
   │
6. 发送推送
   │
   └─→ feishu-reporter 发送周报卡片
```

## 依赖的 Skills

| Skill | 用途 |
|-------|------|
| `storage-helper` | 读取画像、行为日志、推送记录 |
| `profile-updater` | 计算权重变化、准确率 |
| `feishu-reporter` | 发送周报卡片 |
| `arxiv-fetcher` | 查询论文引用情况（可选） |

## 注意事项

1. **推送时间**：周一上午 10 点，避开早高峰
2. **报告长度**：控制在飞书卡片一屏内，过长则折叠
3. **建议数量**：每周建议不超过 5 条，避免信息过载
4. **遗漏检测**：需要外部数据（引用、社交媒体），失败时跳过该部分
