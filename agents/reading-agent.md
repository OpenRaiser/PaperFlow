# Reading Agent

## 职责

精读报告：为选中的论文生成详细分析报告，创建飞书文档并返回链接。

## 触发条件

- 用户从每日推送中选择论文后

## 输入

| 输入 | 来源 | 内容 |
|------|------|------|
| 选中的论文列表 | feedback-agent 传递 | 论文 arxiv_id/doi 列表 |
| 论文全文/摘要 | storage-helper / arxiv-fetcher | 论文元数据、PDF 链接 |

## 输出

### 精读报告文档（飞书文档）

```markdown
# [论文标题]

## 基本信息
- **作者**: [作者列表]
- **机构**: [机构列表]
- **来源**: arXiv:xxxx.xxxxx / DOI:xxx
- **日期**: 2026-04-01

## 一句话总结
[用一句话概括论文的核心贡献]

## 研究背景
[论文要解决什么问题，为什么这个问题重要]

## 核心方法
[论文提出的方法/框架/技术路线，配关键图表]

## 主要结果
[实验结果、对比 baseline、关键指标]

## 创新点
1. [创新点 1]
2. [创新点 2]
3. [创新点 3]

## 局限性
[作者自述或你分析的局限性]

## 与我们研究的关系
[结合用户画像，分析这篇论文对用户研究的价值]
- 可借鉴的方法：...
- 可使用的数据：...
- 可follow 的方向：...

## 代码与资源
- **代码**: [GitHub 链接]
- **数据**: [数据集链接]
- **项目主页**: [项目网站]

## 推荐指数
🔴🔴🔴🟡🟡（3/5）

[推荐理由：为什么值得读/不值得读]
```

### 飞书消息通知

```
📄 精读报告已完成：[论文标题]

链接：https://example.feishu.cn/docx/xxxxx

预计阅读时间：5-8 分钟
```

## 工作流程

```
1. 接收选中的论文列表
   │
   └─→ 从 feedback-agent 获取 selected_papers
   │
2. 获取论文全文
   │
   ├─→ arxiv-fetcher 获取 PDF 链接
   ├─→ pdf-parser 解析 PDF 内容
   └─→ 提取摘要、方法、实验、结论
   │
3. 生成精读报告
   │
   ├─→ 调用 Claude 分析论文内容
   ├─→ 按照模板生成结构化报告
   └─→ 结合用户画像生成"与我们研究的关系"
   │
4. 创建飞书文档
   │
   └─→ feishu-reporter 创建文档并回填内容
   │
5. 发送通知
   │
   └─→ feishu-reporter 发送文档链接
```

## 依赖的 Skills

| Skill | 用途 |
|-------|------|
| `arxiv-fetcher` | 获取论文 PDF 链接 |
| `pdf-parser` | 解析 PDF 全文 |
| `feishu-reporter` | 创建飞书文档、发送通知 |
| `storage-helper` | 读取/更新论文缓存 |

## 报告质量标准

| 维度 | 要求 |
|------|------|
| 准确性 | 方法描述无误，结果数据准确 |
| 可读性 | 结构清晰，语言简洁，配关键图表 |
| 相关性 | "与我们研究的关系" 必须结合用户画像 |
| 完整性 | 包含方法、结果、代码链接等关键信息 |
| 篇幅 | 1000-2000 字，阅读时间 5-8 分钟 |

## 注意事项

1. **并发处理**：用户选择多篇论文时，并行生成报告
2. **进度通知**：每完成一篇立即发送链接，不等待全部完成
3. **PDF 失败处理**：如果 PDF 无法获取，基于摘要生成简化报告
4. **代码检查**：优先检查是否有开源代码，无代码时标注"暂无代码"
