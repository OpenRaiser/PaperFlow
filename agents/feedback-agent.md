# Feedback Agent

## 职责

反馈处理：解析用户在飞书中的回复（编号选择/快捷命令），提取正负信号，记录行为日志，调用 profile-updater 更新画像。

## 触发条件

- 用户回复每日推送卡片的选择消息

## 输入

### 用户回复格式

| 格式 | 示例 | 说明 |
|------|------|------|
| 直接回复编号 | `1 2 4 6 7 9 11` | 选择特定论文 |
| 范围选择 | `1-5 6 9 11` | 选择连续编号 + 特定编号 |
| 快捷命令 | `all lock` | 所有必读清单命中的论文 |
| 快捷命令 | `all red` | 所有高度相关的论文 |
| 快捷命令 | `none` | 今天都不看 |
| 纠正分类 | `16 应该是🔴不是🟡` | 纠正 Agent 分类错误 |

## 输出

### 行为日志记录

```json
{
  "user_id": "user_001",
  "date": "2026-04-08",
  "push_id": "push_20260408",
  "selected": [1, 2, 3, 4, 5, 6, 9, 11, 15],
  "skipped": [7, 8, 10, 12, 13, 14, 16, 17, ..., 47],
  "selection_rate": 0.19,
  "signal_strength": "normal"
}
```

### 飞书确认回复

```
收到，9 篇进入精读队列。
预计 2 小时内全部完成，报告将逐篇发送飞书文档链接。

📊 今日反馈已记录：
  选择：01-06, 09, 11, 15（9 篇）
  跳过：07-08, 10, 12-14, 16-47（38 篇）
  
  学到的信号：
  ✓ 你选了 06(emergence+bio) 但跳过了 08(AutoML+science)
    → 你更关注 "emergence" 而非 "AutoML" 在科学场景的应用？
  ✓ 你选了 15(attention+bio) 虽然它只是 🟡 级
    → 提升 "高效注意力 + 生物序列" 的相关性权重
  ✓ 你跳过了 12(Anthropic constitutional AI)
    → 降低纯 safety/alignment 类研究的推荐权重
    
  以上推断对吗？不用回复，我会根据后续行为自动验证。
  如果某条推断明显错误，可以纠正：
    "不是，08 我只是今天没时间，AutoML for science 我感兴趣"
```

## 信号提取规则

### 正信号（选择了）

| 信号类型 | 权重 | 说明 |
|----------|------|------|
| 选择论文 | 1.0 | 基础正信号 |
| 位置靠后还选了 | 1.2 | 越靠后选了权重越高 |
| 🟡级但选了 | 1.5 | Agent 预测偏低，用户校正 |
| 主动纠正分类 | 2.0 | 强正信号 |

### 负信号（没选择）

| 信号类型 | 权重 | 说明 |
|----------|------|------|
| 跳过普通论文 | 0.3 | 弱负信号（可能只是没时间） |
| 连续 3 天跳过同类 | 1.0 | 强负信号 |
| 主动说"不推这类" | 2.0 | 强负信号 |

### 信号采集维度

```
正信号（选择了）：
├── 论文的主题 embedding
├── 论文的方法类型标签
├── 论文的机构
├── 论文的作者
├── 论文在清单中的位置（越靠后你还选了 → 越强的正信号）
└── 论文的 Agent 预测关联度（Agent 觉得🟡但你选了 → 校正信号）

负信号（没选择）：
├── 同上所有特征
├── 但权重更低（不选择可能是没时间而非不感兴趣）
└── 只有连续多天跳过同类论文才视为强负信号

强信号（用户主动操作）：
├── 你主动纠正： "这个我感兴趣" → 强正
├── 你主动说： "这类以后别推了" → 强负
├── 你修改必读清单 → 直接更新规则
└── 你给精读报告点了 "有用/没用" → 影响报告深度调整
```

## 工作流程

```
1. 接收用户回复
   │
   └─→ 解析消息内容（编号/快捷命令/纠正）
   │
2. 验证选择有效性
   │
   ├─→ 检查编号是否在推送范围内
   └─→ 处理范围选择（如 1-5）
   │
3. 记录行为日志
   │
   └─→ storage-helper 写入 behavior_logs 表
   │
4. 提取信号
   │
   ├─→ 正信号：selected_papers 的特征
   ├─→ 负信号：skipped_papers 的特征
   └─→ 推断用户偏好变化
   │
5. 更新画像
   │
   └─→ profile-updater 更新兴趣向量、主题权重
   │
6. 发送确认回复
   │
   └─→ feishu-reporter 发送反馈确认卡片
```

## 依赖的 Skills

| Skill | 用途 |
|-------|------|
| `storage-helper` | 记录行为日志 |
| `profile-updater` | 更新用户画像 |
| `feishu-reporter` | 发送确认回复 |

## 注意事项

1. **信号权重**：正信号权重 > 负信号权重（不选择可能是没时间）
2. **漂移检测**：连续 3 天跳过同类论文视为兴趣下降
3. **推断验证**：生成的推断不需要用户确认，但允许用户纠正
4. **繁忙日处理**：用户回复 `all lock` 时，不更新偏好模型（信号不可靠）
