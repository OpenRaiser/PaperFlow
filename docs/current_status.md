## 二、已知问题

### 2.1 数据层问题

| 问题 | 严重程度 | 状态 | 描述 |
|------|----------|------|------|
| 论文去重逻辑缺陷 | 🔴 高 | ✅ 已修复 | `save_paper` 只检查 arxiv_id，导致无 arxiv_id 的论文全部重复 |
| 无 arxiv_id/doi 论文落到同一 paper_id | 🔴 高 | ✅ 已修复 | 缺失标识时写入空字符串，触发 `papers.arxiv_id` 唯一约束后回退到同一条旧记录 |
| behavior_logs 数据冗余 | 🟡 中 | ⚠️ 部分修复 | 同一论文有多条记录（pushed/selected/skipped），导致 JOIN 重复 |
| get_latest_push 排序错误 | 🟡 中 | ✅ 已修复 | 使用 timestamp 排序而非 rank，导致编号映射错乱 |
| category 字段未保存 | 🟡 中 | ✅ 已修复 | 推送时动态计算的 category 未保存，反馈时无法识别 |

### 2.2 逻辑层问题

| 问题 | 严重程度 | 状态 | 描述 |
|------|----------|------|------|
| all red 选择错误 | 🔴 高 | ✅ 已修复 | 选择连续编号 1-N 而非实际 high_relevant 论文 |
| all lock 选择错误 | 🔴 高 | ✅ 已修复 | 同上 |
| 反馈编号硬编码上限 50 | 🔴 高 | ✅ 已修复 | roleA 实际推送可超过 50 篇，导致 `51-60` 等编号无法识别 |
| 反馈后重复推送 | 🔴 高 | ✅ 已修复 | webhook 会把 bot 自己发出的 `📰/📊/📋` 消息当成用户消息再次路由，现已补内容过滤并重启为单实例 |
| 推送与反馈数据不同步 | 🔴 高 | ⚠️ 部分修复 | 已修复 push 取数只读取 `action='pushed'` 且按 `rank` 还原顺序，历史脏数据仍需清理 |
| 画像修正语句无法生效 | 🟡 中 | ✅ 已修复 | `我最近对 X 更感兴趣了` / `降低 GUI Agent 权重` 现在会路由到 `profile_update` 并真正更新画像 |

### 2.3 交互层问题

| 问题 | 严重程度 | 状态 | 描述 |
|------|----------|------|------|
| Bot 消息过滤不完整 | 🔴 高 | ✅ 已修复 | 补充了 `📰 今日论文`、`📋 你的学术画像`、`收到，`、`📊 今日反馈已记录` 等回声消息过滤规则 |
| 飞书推送时机不一致 | 🟡 中 | ⚠️ 待观察 | 用户看到推送与数据库记录可能不是同一批 |
| `none` 反馈被误判为无效 | 🟡 中 | ✅ 已修复 | 现在会作为合法“今日都不看”反馈处理并正常记录 skipped |
| 角色 chat_id 未同步到 profile | 🟡 中 | ✅ 已修复 | `daily-push-agent` 现在会回退读取 `data/roles.json`，避免多角色推送落到默认 open_id |
| 多个旧 webhook 进程残留 | 🔴 高 | ✅ 已处理 | 旧进程仍在执行旧逻辑，已清理并重启为单进程 |
| 冷启动仍显示旧版纯文本模板 | 🟡 中 | ✅ 已修复 | `coldstart-agent` 存在重复函数定义，旧模板覆盖新模板；已删除覆盖逻辑并统一到 PDF 风格 |

## 2026-04-12 补充

- 日推链路已从“标题哈希伪 embedding”切到统一 embedding service。
- `OpenReview` 未配置时默认不再返回 `Sample Paper` 假数据，只有显式开启 `SCITASTE_ALLOW_MOCK_PAPERS=true` 才会生成 mock。
- embedding 现支持 `hash / openai / local` 三种 provider，默认 `hash`，维度默认固定为 `768`。
- `OpenReview` 抓取已修正为使用真实 submission invitation，不再错误调用 `venue=`；client 会在进程内缓存，避免多会场抓取时反复登录触发限流。
- `fetch_openreview.fetch_by_date()` 现在会按日期范围推断会议年份，并过滤超出范围的会议信息，避免 2026 日推误抓 2024 论文。
